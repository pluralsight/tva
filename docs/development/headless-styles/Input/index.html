Input

caution

This is unreleased documention for the experimental version of the headless-styles package.

The getInputProps helper is used to get user input in a text-like field.

View source

Import‚Äã

import { getInputProps } from '@pluralsight/headless-styles'

Usage‚Äã

Basic Input‚Äã

const { fieldOptions } = getFormControlProps()
const inputProps = getInputProps({
  ...fieldOptions,
  id: 'basic-input',
  name: 'basic_input',
  placeholder: 'Basic input',
  value: ''
})

<input {...inputProps} onChange={handleCheck} />

Sizes‚Äã

The getInputProps helper comes with two sizes: m and l (default).

const { fieldOptions } = getFormControlProps()
const mInputProps = getInputProps({
  ...fieldOptions,
  id: 'basic-input',
  name: 'basic_input',
  placeholder: 'Basic input',
  size: 'm',
  value: ''
})
const inputProps = getInputProps({
  ...fieldOptions,
  id: 'basic-input',
  name: 'basic_input',
  placeholder: 'Basic input',
  size: 'l',
  value: ''
})

<input {...mInputProps} onChange={handleCheck} />
<input {...inputProps} onChange={handleCheck} />

Password Input Example‚Äã

You can combine helpers to create a password input with show/hide functionality:

import { useState } from 'react'
import {
  getButtonProps,
  getFormControlProps,
  getIconProps,
  getInputProps,
} from '@pluralsight/headless-styles'
import { EyeIcon, EyeOffIcon } from '@pluralsight/icons'

const fieldContainerStyle = {
  position: 'relative',
  width: '100%',
}

const btnStyle = {
  position: 'absolute',
  right: '0.3rem',
  top: '12px',
  zIndex: '100',
}

function PasswordInput() {
  const [show, setShow] = useState(false)
  const [password, setPassword] = useState('')
  const { fieldOptions } = getFormControlProps()
  const inputProps = getInputProps({
    ...fieldOptions,
    id: 'password',
    name: 'user_password',
    placeholder: 'Basic input',
    type: show ? 'text' : 'password',
    value: password,
  })

  function handleChange(e) {
    setPassword(e.target.value)
  }

  function handleToggleShow() {
    setShow((prev) => !prev)
  }

  return (
    <div style={fieldContainerStyle}>
      <input {...inputProps} onChange={handleChange} />
      <button {...getButtonProps({ kind: 'weak', size: 's' })} style={btnStyle}>
        <span {...getIconProps({ size: 'm' })} onClick={handleToggleShow}>
          {show ? <EyeIcon /> : <EyeOffIcon />}
        </span>
      </button>
    </div>
  )
}

export default PasswordInput

Accessibility and messages‚Äã

When displaying Error Messages or Help Text it is required to pass the options.id value to the InputOptions.describedBy field. This will add a aria-describedby attribute to the input which establishes a relationship between the input and message you are displaying.

danger

Not providing a describedBy option that matches the Error/Field Message id will result in a poor accessbility experience. Please make sure you are using both of these in order to provide the best experience to all of your users.

Using both error & field messages‚Äã

When using both the field message and error message API, pass a comma-separated String to the InputOptions.describedBy option.

Example using both messages for an input
const { value: helpText, ...message } = getFieldMessageProps({
  id: 'email:help',
  message: 'We will never share your email.',
})
const error = getErrorMessageProps({
  id: 'bad:email',
  invalid,
  message: 'An email address is required.',
})
const inputProps = getInputProps({
  ...fieldOptions,
  describedBy: `${message.id},${error.container.id}`,
  id: 'email',
  name: 'user_email',
  value,
})

Options‚Äã

InputOptions‚Äã

This interface is available as an import for use.

interface InputOptions extends FieldOptions {
  describedBy?: string
  id: string
  name: string
  placeholder: string
  size?: 'm' | 'l'
  tech?: 'svelte'
  type: 'text' | 'email' | 'password' | 'tel' | 'url'
  value: string
}
Import example
import type { InputOptions } from '@pluralsight/headless-styles/types'

FieldOptions‚Äã

interface FieldOptions {
  disabled?: boolean
  invalid?: boolean
  readOnly?: boolean
  required?: boolean
}
