Icon

caution

This is pre-release documention for the experimental version of the headless-styles package.

Icons are used to convey meaning at a glance.

View source

Import‚Äã

import { getIconProps } from '@pluralsight/headless-styles'
  • getIconProps: The styles and a11y attributes for any SVG icon

Usage‚Äã

Icon sizes‚Äã

Use the size options to change the size of an icon. The options are: s, m (default), l.

<StarIcon {...getIconProps({ size: 's' })} />
<StarIcon {...getIconProps({ size: 'm' })} />
<StarIcon {...getIconProps({ size: 'l' })} />
tip

We use the standard clothing size shorthand values for all size related options in any library we provide (i.e. xs-xxl).

Accessibility‚Äã

An icon alone does not convey any useful information to non-visual users. Depending on the usage, you will need to provide additional information.

Decorative icons‚Äã

The icon is purely cosmetic and conveys no real semantic meaning. In this case, the icon should be hidden from assistive technology.

¬†Business Account
<span>
  <BriefcaseIcon {...getIconProps({ ariaHidden: true })} /> Business Account
</span>

Interactive icons‚Äã

SVG elements are not semantically interactive elements; when the user would interact with an icon, a more appropriate element should "own" the interaction:

  • The icon element should be a child of a <button> or <a> element.
  • The button or a tag should have a meaningful label provided either through direct text content, aria-label, or aria-labelledby.
Page 1Page 2Page 3Page 4
Labelling with aria-label
import { getIconProps, getIconButtonProps } from '@pluralsight/headless-styles'
import { HomeIcon } from '@pluralsight/icons'

const { button, iconOptions } = getIconButtonProps({
  ariaLabel: 'Navigate to home page',
})

function HomeButton() {
  return (
    <a className={button.className} href="/">
      <HomeIcon {...getIconProps(iconOptions)} />
    </a>
  )
}
Labelling with aria-labelledby
import { getIconProps, getButtonProps } from '@pluralsight/headless-styles'
import { HomeIcon } from '@pluralsight/icons'

const buttonProps = getButtonProps()

function HomeButton() {
  return (
    <a aria-labelledby="homeLink" className={buttonProps.className} href="/">
      <HomeIcon {...getIconProps({ ariaHidden: true })} />
      <span id="homeLink" style={{ display: 'none' }}>
        Navigate to home page
      </span>
    </a>
  )
}
Labelling with direct text
import { getIconProps, getButtonProps } from '@pluralsight/headless-styles'
import { HomeIcon } from '@pluralsight/icons'

const buttonProps = getButtonProps()

function HomeButton() {
  return (
    <a className={buttonProps.className} href="/">
      <HomeIcon {...getIconProps({ ariaHidden: true })} />
      <span className="visually-hidden">Navigate to home page</span>
    </a>
  )
}
tip

We provide iconOptions for you to pass into the getIconProps helper if you are adding icons into a button with text. The iconOptions will pass all necessary a11y settings on your behalf to getIconProps.

Indicator icons‚Äã

When an icon visually communicates some information (such as a status), that information must also be made available to screen readers. Use the ariaLabel option to add a non-visible, accessible label to the icon.

<VolumeOffIcon {...getIconProps({ ariaLabel: 'Volume is off' })} />

IconOptions‚Äã

Guides‚Äã

Icon size guide‚Äã

Icon sizes do not have any relativity to any other elements. Here is an interface that gives the sizing guide.

Example interface - does not exist in types
interface IconSizeGuide {
  s: {
    height: '1rem'
    width: '1rem'
  }
  m: {
    height: '1.25rem'
    width: '1.25rem'
  }
  l: {
    height: '1.5rem'
    width: '1.5rem'
  }
}
