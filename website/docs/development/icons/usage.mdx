# Usage (Web)

:::caution

This is **experimental** documentation for Pluralsight Design **icons** package.

:::

The Icons package provides inline SVG icons as ready-to-use components for popular frameworks as well as SVG files and data URIs.

## Import

We export React components by default, which can be imported by name from the package root.

### React

```javascript
import { MenuIcon } from '@pluralsight/icons'
```

### Svelte

```javascript
import { MenuIcon } from '@pluralsight/icons/svelte'
```

### SVG

```javascript
import menuSvg from '@pluralsight/icons/svg/navigation/menu.svg'
```

### Typescript and pathed imports

To use imports other than the default (`@pluralsight/icons`)
or direct build file references with Typescript you will need
**version 4.5 or greater** and
[moduleResolution](https://www.typescriptlang.org/tsconfig#moduleResolution) set to `nodenext`

```json title="tsconfig.json"
{
  "compilerOptions": {
    "moduleResolution": "nodenext"
  }
}
```

If you are unable to upgrade or use these settings, you can reference the build files directly.

### Data URI

If you need to embed an icon in an `img` tag or using for a CSS property value, you can use the `getIconString` method.

```jsx title="Example using an img src value"
import { getIconString } from '@pluralsight/icons/string'

function MenuIcon(props) {
  return (
    // highlight-next-line
    <img alt="menu icon" src={getIconString('MenuIcon', props.color)} />
  )
}
```

Here is an example using `getIconString` as a CSS value.

```jsx title="Example using a CSS property value"
import { getIconString } from '@pluralsight/icons/string'

function MenuIcon(props) {
  return (
    <span
      aria-label="menu icon"
      style={{
        /* highlight-next-line */
        backgroundImage: `url("${getIconString('MenuIcon', props.color)}")`,
        display: 'inline-block',
        height: '1rem',
        width: '1rem',
      }}
    ></span>
  )
}
```

### Icon colors

By default the icons are monochromatic and will inherit the `color` of their parent.
This ensures the icon color matches its context by default and facilitates
theming while still allowing for control over the icon's color directly.

<IconColor />

```jsx
<button
  style={{
    backgroundColor: 'var(--ps-background-weak)',
    borderRadius: '6px',
    color: 'var(--ps-text)',
  }}
>
  <PlaceholderIcon {...getIconProps()} />
  Icon matches font color
</button>
```

## Icon size guide

We have defined four standard sizes for icons. This sizing should be applied using the [headless-styles](../headless-styles/Icon.mdx#icon-sizes) package.

```typescript title="Example interface - does not exist in types"
interface IconSizeGuide {
  s: {
    height: '1rem'
    width: '1rem'
  }
  m: {
    height: '1.25rem'
    width: '1.25rem'
  }
  l: {
    height: '1.5rem'
    width: '1.5rem'
  }
}
```
